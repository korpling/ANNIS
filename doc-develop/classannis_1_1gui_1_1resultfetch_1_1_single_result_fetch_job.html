annis.gui.resultfetch.SingleResultFetchJob Class Reference

Detailed Description

Fetches a result which contains only one subgraph.

This single query always follows a normal ResultFetchJob and so it is assuming that there already exists a list of matches. That is the reason for not needing to execute the find command and hopefully this query is bit faster.
